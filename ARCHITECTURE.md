# grnet_eip (企業資訊入口網站) 專案架構說明

這是一份用於記錄並維護 `grnet_eip` 系統流程與架構規劃的文件。本專案為公司內部使用的企業資訊入口網站。

## 1. 技術棧規劃

*   **前端框架**：Vue.js + Tailwind CSS
    *   負責畫面渲染、UI 樣式與使用者互動。
*   **後端框架**：Node.js (建議搭配 Express.js)
    *   **架構師建議**：Vue 本身是前端框架，無法直接處理後端伺服器與資料庫層。為了讓技術棧單一化，建議採用同為 JavaScript 生態系的 Node.js。這樣前後端語言統一，方便團隊協作、後續維護與全端開發。
*   **資料庫**：JSON Mock Data
    *   初期暫無正式資料庫，以 Node.js 讀寫 JSON 檔案來替代，待功能確認與業務邏輯穩定後，再移轉至關聯式資料庫。

## 2. 系統權限與角色 (RBAC 規劃)

系統未來需建立完善的權限與視圖機制，目前定義的組織編制與角色如下：

*   **管理層 (Admin)**
    *   老闆 (2人) - 擁有最高權限、可審核及檢視全域數據。
*   **員工層 (Staff)**
    *   客服專員 (3人)
    *   企劃專員 (4人)
    *   設計師 (4人)
    *   前端開發工程師 (2人)
    *   後端開發工程師 (4人)

## 3. 核心功能模組 (初期規劃)

目前系統主要針對公司內部日常需求進行開發，預計包含以下三大功能板塊：

1.  **差勤請假系統**
    *   員工發起請假申請。
    *   配合不同角色權限的請假單審核流程機制。
2.  **員工排程與行事曆**
    *   每位員工的工作排程、會議時間建立。
    *   跨部門協作與個人的行事曆檢視功能。
3.  **客戶資料管理**
    *   客戶基本資料建立與聯絡維護。

*(後續新功能與架構異動將依需求逐步新增於此文件中)*
