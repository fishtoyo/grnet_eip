# grnet_eip (企業資訊入口網站) 專案架構說明

這是一份用於記錄並維護 `grnet_eip` 系統流程與架構規劃的文件。本專案為公司內部使用的企業資訊入口網站。

## 1. 技術棧規劃

*   **前端框架**：Vue.js + Tailwind CSS
    *   負責畫面渲染、UI 樣式與使用者互動。
*   **後端框架**：Node.js (建議搭配 Express.js)
    *   **架構師建議**：Vue 本身是前端框架，無法直接處理後端伺服器與資料庫層。為了讓技術棧單一化，建議採用同為 JavaScript 生態系的 Node.js。這樣前後端語言統一，方便團隊協作、後續維護與全端開發。
*   **資料庫**：JSON Mock Data
    *   初期暫無正式資料庫，以 Node.js 讀寫 JSON 檔案來替代，待功能確認與業務邏輯穩定後，再移轉至關聯式資料庫。

## 2. 系統權限與角色 (RBAC 規劃)

系統未來需建立完善的權限與視圖機制，目前定義的組織編制與角色如下：

*   **老闆 (管理層)**：擁有最高權限。行事曆可檢視全公司/各部門所有人的工作排程，並具備篩選功能（如：只看企劃）。
*   **會計 (財務部)**：負責薪資與帳務。專屬功能包含：薪資計算、員工年資列表、公司收帳等。
*   **企劃 (專案部)**：負責專案推進與工程排程。專屬功能包含：建立客戶專案資料、修改單建立。其行事曆主要用於安排與客戶溝通開會，並負責建立工程部門的每日排程。
*   **客服 (客服部)**：負責客戶第一線應對。其行事曆主要用於記錄與客戶的互動或會議。
*   **視覺設計師、前端工程師、後端工程師 (工程部)**：負責專案執行。專屬功能包含：我的專案、排程清單。其行事曆視角為「工程排程」，由企劃統一建立分派。

## 3. 核心功能模組 (初期規劃)

目前系統主要針對公司內部日常需求進行開發，預計包含以下三大功能板塊：

1.  **差勤請假系統**
    *   員工發起請假申請。
    *   配合不同角色權限的請假單審核流程機制。
2.  **全域行事曆與排程系統 (依權限動態渲染)**
    *   企劃/客服：客戶會議與行程追蹤。
    *   工程/設計：檢視被指派的工程排程。
    *   老闆：全域視圖與部門篩選。
3.  **財務與人事管理**
    *   會計專用的薪資結算、公司帳務收取與年資紀錄。
4.  **專案與客戶管理**
    *   企劃專用的客戶資料建立、修改單追蹤，以及工程人員的專案清單。

## 4. 版本控制 (Version Control)

*   **GitHub Repository**：`https://github.com/fishtoyo/grnet_eip.git`
*   **日常更新同步指令**：
    1. `git add .` (將所有變更檔案加入暫存區)
    2. `git commit -m "簡述這次新增或修改的功能"` (提交變更)
    3. `git push` (推送到 GitHub 遠端資料庫)

## 5. 專案目錄結構與啟動方式

本專案採用前後端分離開發，但放置於同一個 Repository 管理：

*   `/frontend`：前端 Vue 專案目錄
    *   啟動方式：`cd frontend` -> `npm run dev`
*   `/backend`：後端 Node.js (Express) 專案目錄
    *   啟動方式：`cd backend` -> `node server.js`

*(後續新功能與架構異動將依需求逐步新增於此文件中)*

## 6. 開發規範與專案日記

### 開發規範
1. **目錄建立規則 (嚴格遵守)**：若開發步驟中需要「新增資料夾」，AI 助手會優先暫停所有後續動作與程式碼生成，並請開發者先手動把資料夾建立完成。待開發者確認建立後，才會接著提供下一步驟。
2. **樣式開發規則**：本專案為 Vue 與 Tailwind CSS 搭配，所有的畫面佈局與樣式一律優先使用 Tailwind CSS 處理，並**需使用 Tailwind CSS 最新版本寫法**。盡量避免額外撰寫自訂的 CSS（即盡量保持 `.vue` 檔內無 `<style>` 區塊或維持最少自訂樣式）。

### UI 設計規範 (Design System)
1. **內頁版面 (Layout)**：預設內頁採滿版寬設計，外層容器統一標準樣式：`bg-white p-6 md:p-8 rounded-xl shadow-sm border border-gray-100 min-h-[calc(100vh-4rem)]`。
2. **按鈕元件 (Buttons)**：統一形狀、間距與互動效果，基礎樣式為 `px-5 py-2.5 font-medium rounded-lg transition-colors shadow-sm`。依主次要行為更換顏色（例如主要：`bg-gray-800 text-white`，次要/返回：`bg-gray-100 text-gray-700`）。
3. **列表呈現 (Lists)**：資料條目摒棄卡片式 (Card) 造型，改採滿版橫列 (Row) 佈局，條目間以底線 (`border-b border-gray-100`) 分隔，並加上 `hover:bg-gray-50` 提升互動感。
4. **響應式設計 (RWD)**：所有畫面佈局與元件皆須搭配 RWD 處理，確保在手機 (Mobile)、平板 (Tablet) 與電腦 (Desktop) 等不同尺寸螢幕下皆能維持良好的閱讀體驗與操作性（善用 Tailwind 的 sm:, md:, lg: 等斷點）。
